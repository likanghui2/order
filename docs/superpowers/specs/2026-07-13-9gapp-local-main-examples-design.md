# 9G App 本地 Main 示例设计

## 目标

为 `task/9Gapp/search.py` 和 `task/9Gapp/sham_booking.py` 增加可直接运行的本地请求示例，方便开发者手工联调 9G 搜索和单 PNR 押位流程。

## 设计

- 两个文件都在末尾增加 `if __name__ == "__main__":` 入口。
- 搜索示例使用当前任务协议的完整 `search` 请求，来源为 `9GAPP`，示例航线为 `SGN-PQC`。
- 押位示例使用当前任务协议的完整 `shamBooking` 请求，示例航班为 `9G0123`、舱位为 `Y`、产品为 `ECONOMY LITE`。
- 示例直接调用各自的 Celery `main` 任务，与仓库已有航司任务的本地调试方式保持一致。
- 代理结构完整，但用户名、密码、主机等敏感值使用明显占位符，不提交真实凭据。
- 注释明确说明运行押位示例前必须用实时搜索结果更新日期、航班号、舱位和产品。

## 错误与安全

- 占位代理未替换时，真实请求预期失败；示例不绕过现有参数校验。
- 不捕获或吞掉异常，保留当前任务装饰器和日志行为。
- 不修改生产任务调用路径、模型、搜索或押位逻辑。

## 验证

- AST/编译检查确保两个 `__main__` 入口语法有效。
- 单元测试检查两个文件都含完整的本地请求入口，且示例来源和任务类型正确。
- 运行现有完整 pytest 套件，确保导入模块时不会执行真实请求。
